项目概述
这是一个闲鱼（咸鱼）商品智能监控机器人，核心思路是：用 Playwright 模拟真人浏览器操作闲鱼网站 → 抓取商品数据 → AI 分析是否值得买 → 推送通知。

爬虫工作流程（从 spider_v2.py 到 src/scraper.py）
1. 入口：任务编排（spider_v2.py）
spider_v2.py 是 CLI 入口，负责：

从数据库（或 config.json）加载所有已启用的监控任务
每个"任务"包含：搜索关键词、价格范围、AI prompt 文件、cron 调度等
支持参数筛选：--task-name 只跑单个任务、--debug-limit 限制处理商品数
用 asyncio.gather 并发执行多个任务
2. 核心：浏览器爬取（src/scraper.py 的 scrape_xianyu()）
这是最核心的函数，整个流程如下：

步骤 0 — 模拟真人反检测

启动 Chromium 浏览器（非 headless 模式可手动处理验证码）
  ├── 伪装成移动端 Nexus 5 设备（UA、viewport、触摸、地理位置全模拟）
  ├── 注入反检测脚本（移除 webdriver 标识、伪造 plugins/languages/chrome 对象）
  ├── 加载已保存的登录态（cookies/state.json）
  └── 先访问 goofish.com 首页，模拟真人浏览（随机滚动、随机等待）
关键反爬措施：

移动端 UA 伪装（scraper.py:251-260）
注入 JS 移除 navigator.webdriver（scraper.py:611-632）
每步操作间都有 random_sleep 随机延迟，模拟人类节奏
支持增强版浏览器快照（保留原始 headers、屏幕参数等）
步骤 1 — 搜索 + 筛选

导航到搜索页 https://www.goofish.com/search?q=关键词
  → 拦截搜索 API 响应（mtop.taobao.idle.pc.search）获取 JSON 数据
  → 应用筛选条件：
     - "新发布"排序
     - "个人闲置"过滤
     - "包邮"过滤
     - 区域筛选（省/市/区三级联动点击）
     - 价格区间（填写最低/最高价）
  → 每次筛选都会重新拦截 API 响应
步骤 2 — 遍历商品列表

解析搜索 API 返回的 JSON → 得到商品列表
  → 对每个商品：
     ├── 去重（通过链接 unique key，跳过已处理过的）
     ├── 打开商品详情页（新 tab）
     ├── 拦截详情 API 响应获取完整数据：
     │    - 商品标题、价格、图片列表
     │    - 卖家信息（注册天数、芝麻信用）
     │    - "想要"人数、浏览量
     ├── 提交到后台分析队列（ItemAnalysisDispatcher）
     ├── 随机等待 5-10 秒（模拟看商品的时间）
     └── 关闭详情页，处理下一个
步骤 3 — AI 分析 & 通知
每个商品被提交到 ItemAnalysisDispatcher（后台并发队列），进行：

卖家画像采集：访问卖家个人主页，爬取其商品列表和评价（scrape_user_profile）
价格参考：与历史价格快照对比，判断是否低价
AI 分析（两种模式）：
ai 模式：把商品信息 + 图片 + 卖家信息 + prompt 发给多模态 AI 模型
keyword 模式：纯关键词匹配，不走 AI
通知推送：符合条件的商品通过 ntfy/Bark/企业微信/Telegram/Webhook 推送
整体调度架构

Web UI 创建任务
  ↓
TaskService 存储 → SQLite
  ↓
SchedulerService (APScheduler) 按 cron 表达式定时触发
  ↓
ProcessService 启动 spider_v2.py 子进程
  ↓
爬取 → AI 分析 → 通知推送
  ↓
结果存储到 jsonl/ 目录 + 数据库
其他亮点
账号/代理轮换：支持多账号和多代理 IP 轮换，失败时自动切换（scraper.py:184-234）
失败保护（FailureGuard）：连续失败 N 次自动暂停任务，修复后自动恢复
风控检测：检测到验证弹窗（baxia-dialog）或 FAIL_SYS_USER_VALIDATE 立即停止，避免账号被封

步骤 3 — 逐商品打开详情页抓取

整个流程就是：Playwright 模拟浏览器操作 → 拦截 API 响应拿 JSON → 解析字段 → 提交给下游分析。不依赖页面 DOM 解析，而是直接从 API 响应中提取结构化数据。
